# Findings & Decisions

## Requirements
- 평면 모니터 3D 게임(FPS류) 플레이 중 멀미/불편함 완화
- 머리 회전(=화면 카메라 회전) 방향에 맞춘 방향성 진동 제공
- 개인이 제작 가능한 저비용 구성
- 제작자는 아두이노/전자공작 **초보**
- 대상은 VR이 아님 (사용자는 VR에선 멀미를 느끼지 않음)

## Research Findings
선행 연구 조사 결과 (출처는 Resources 참고):

- **멀미 원인**: 시각-전정 감각 충돌(sensory conflict). 눈은 "움직인다", 전정기관은 "정지"로 인식 → 충돌.
- **개입 방식 두 갈래 (메커니즘이 반대)**:
  - (a) **노이즈 주입** (GVS·골전도 진동): 전정 신호 신뢰도를 일부러 낮춰 뇌가 시각에 가중치를 더 둠.
  - (b) **방향 일치 진동 큐**: 움직임 방향에 맞춰 인공 자기운동 단서를 보충. ← **본 프로젝트의 접근**
- **근거 강도 비교 (조사 시점 결론)**:
  - 노이즈 GVS (Exp. Brain Research, 2020): 효과가 **일시적**(자극 중단 후 3~6분 소멸), 강한 VR+구두척도에서만 유의, 표준 설문에선 무의.
  - 골전도 진동 (PLOS One 2018 / Frontiers in VR 2024, n≈30): **탐색적**, 표본 작음, 긍정적 경향이나 검정력 낮음.
  - WalkingVibe (CHI 2020, **n=240**): 멀미·불편 **유의 감소** — 방향성 진동의 가장 강한 근거. 단 **"걷기" 시나리오 한정**.
  - InertiaVibe (2022): 관성/회전 방향 진동 — **사용자 아이디어와 가장 근접**하나 표본 작고 주로 주관적 보고.
- **핵심 한계**: 진동 개입 연구는 대부분 **VR/시뮬레이터** 대상. **평면 3D 게임 + 방향 진동**은 거의 미개척 → 본 프로젝트는 단순 재현이 아닌 **탐색적 실험**.
- **모터 위치 시사점**: 전정 자극을 노린다면 **귀 뒤 유양돌기(mastoid)** 근처가 근거상 합리적. 목/어깨는 "촉각 그라운딩"에 가까움.
- **자극 주파수 (중요 — 가정 수정)**: 골전도 전정 자극에 근거 있는 주파수는 깊은 저음이 아니라 **~500Hz**.
  - 임상 전정검사(VEMP) 표준 자극이 500Hz 골전도 톤버스트.
  - vection 연구(Sci Rep 2023): 여러 주파수 중 **500Hz BCV가 전정 노이즈로 가장 효과적**, 자기운동감 가장 잘 유발.
  - → 앞서 가정한 "50~100Hz 필요"는 부정확. 500Hz는 싸구려 엑사이터·일반 골전도 이어폰으로 충분히 출력 가능.
- **하드웨어 저주파 한계**: 소비자 골전도 이어폰은 60~100Hz 이하 급감, "저음 보장 모델" 사실상 없음. DIY 엑사이터(Dayton BCE-1)도 **300Hz~19kHz**, 저주파 THD 높음. 50Hz급 깊은 저음은 택타일 셰이커급 필요(머리 착용 부적합).
  - 결론: **타겟 주파수를 ~500Hz로 잡으면** 평범한 골전도 이어폰 또는 Dayton BCE-1+앰프로 시작 가능 → "저주파 보장" 부담 해소.
- **주파수 대역 폭 (정밀화 — 좁지 않음)**: 500Hz는 '피크'지 좁은 점이 아님.
  - 톤: 임상 VEMP는 **250–750Hz**(고령층 피크 750–1000Hz로 이동)에서 시행, 다중주파수가 더 많이 반응 잡음 → 반응 대역 **~250–1000Hz(약 2옥타브)**.
  - 노이즈: stochastic resonance 연구는 **1–4000Hz 평탄 대역 노이즈** 사용 → 주파수 정밀도 무의미.
  - 함의: **주파수 정밀도는 삽질 리스크가 아님.** 트랜스듀서(300Hz~)가 대역 커버. SW로 톤(500/700/900Hz)·노이즈를 스윕하며 탐색.
  - **진짜 변수 (이게 핵심)**: ① 소형 트랜스듀서의 출력(force)이 충분한가, ② 평면 게임 멀미로 효과가 전이되는가. (주파수 아님)
- **장착/압착력 (설계 핵심 — 2026-06-09 정립)**: 골전도 전달은 **정적 접촉압(클램프력)** 에 크게 의존. 위치보다 "얼마나 단단히 뼈에 눌리느냐"가 효과를 가른다.
  - 두개골은 통째로 진동 → **정확한 유양돌기 정중앙은 불필요**(임상 VEMP도 유양돌기 또는 이마 Fz 사용). 타깃 = 귀 뒤·위 단단한 뼈에 단단히 접촉.
  - 마운트 결론: **오버이어 헤드폰 도너**의 스프링 밴드 클램프력 활용. 쿠션 링의 뒤-아래가 유양돌기를 지남 → 그 지점 쿠션을 도려내고 트랜스듀서를 쿠션면보다 살짝 **돌출**시켜 힘을 집중. (양면테이프=위치만, 압착 부족.)

## Technical Decisions
| Decision | Rationale |
|----------|-----------|
| 회전 소스 = **마우스(주) + 컨트롤러(부)** | 게임 비종속·저지연. 마우스 우선 구현, 컨트롤러는 공통 인터페이스로 대응 |
| **두 트랙 병행** (A=전정/이어폰, B=촉각/모터) | "전정 vs 촉각" 미결 질문을 3way 검증으로 직접 비교 |
| **공통 코어 + 출력 백엔드 2개** | 입력·매핑(→ `(left_level,right_level)`)은 한 번만, 출력만 오디오/시리얼로 분기 |
| Track A 타겟 = ~500Hz 중심(반응대역 250–1000Hz) | 톤은 250–1000Hz 넓게 반응(피크 500), 노이즈는 1–4kHz 광대역. 주파수 정밀도는 리스크 아님 |
| Track A 신호 경로 = PC 오디오 → 별도 출력 장치 | `sounddevice` device 지정. 게임=기본 장치와 분리 |
| 검증 = OFF/전정/촉각 3way 블라인드 | 두 메커니즘을 같은 잣대로 비교 |
| Track A 트랜스듀서 = 소형 8Ω 골전도(Adafruit #1674 또는 동일 제네릭) ×2 | 작고(9.6g)·8Ω·선 달려나옴·머리 착용 적합. 약하면 더 큰/센 것으로 업그레이드 |
| Track A 구동 옵션 (미확정) | (1) PC 오디오→PAM8403 앰프, 또는 (2) 아두이노+DRV8833로 500Hz 생성. 후자는 산 아두이노 활용·모터와 리그 공용·납땜↓ |
| MCU = Arduino Uno + USB 시리얼 | 초보 자료 풍부, 유선이 무선보다 저지연 |
| 액추에이터 = 코인 ERM 모터 | 저렴·단순, POC에 적합. 고급화 시 LRA+DRV2605L |
| 구동 = 트랜지스터+다이오드(채널당) | 핀 직결 불가. 대안: DRV8833 모듈(배선 ↓) |
| 모터 배치 = 귀 뒤 헤드밴드 | 전정 자극 가설 부합 |
| 개발 순서 = LED로 먼저 연습 후 모터 | 초보 리스크 최소화, 단계별 검증 |
| Track A 마운트 = **오버이어 헤드폰 도너**(베어 트랜스듀서 이식) | 밴드 스프링 = 강·일정·조절 가능한 클램프력. 링 뒤-아래가 유양돌기 통과 |
| 결합 = 쿠션 도려내고 트랜스듀서 **돌출**, 피부 사이 푹신폼 금지 | 분산·댐핑 방지, 클램프력 집중. 벨크로로 위치 튜닝 |
| 폴백 2순위 = 골전도 이어폰 프레임 재사용 | 1순위 실패 시. 귀 기준 위치 안정 |
| Stage 0 사전 게이트(테이프+손압착 지도, 아두이노+DRV8833 500Hz) | 만들기 전 "느껴지나/최적 위치/힘 vs 출력" 진단 |

## Issues Encountered
| Issue | Resolution |
|-------|------------|
| 초기 논의가 VR 맥락으로 흘러감 | 사용자 정정: 대상은 평면 3D 게임. 데이터 소스를 HMD→입력장치로 전환 |
| 유선 골전도 이어폰 완제품 희소(시장 BT-only) | 트랜스듀서 베어부품 자작으로 전환 |
| 알리 "bone conduction" 키워드 낚시 — 실제론 일반 스피커/이어폰 드라이버 다수 | 제목 무시. Type에 扬声器/loudspeaker·IEC-711·SPL@1kHz면 탈락. 骨传导/transducer + 두꺼운 금속 원반만 채택. 확실히는 국내 정품(디바이스마트 등) |
| 온이어 컵은 귀 "위"에 앉아 유양돌기를 비껴감 | 오버이어로 전환 — 쿠션 링 뒤-아래가 유양돌기 통과. 트랜스듀서 돌출로 압착 집중 |
