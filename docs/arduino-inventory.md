## 5. 원본 BOM 대비 메모
- **개별 트랜지스터(2N2222)·플라이백 다이오드(1N4148) 불필요** — DRV8833이 H-브리지+보호를 내장. `hardware-design.md §B.2`의 권장 경로를 그대로 충족하며, 초보 최다 실수(다이오드 방향·배선)를 회피.
- **능동 부저는 고정 톤만** → `tone()` 주파수 실험은 **트랜스듀서 + DRV8833**로 진행(커리큘럼 5단계). 부저는 단순 on/off 알람 연습용.
- **코인모터 스펙 미확인** → 3V로 가정. 5V 구동 시 PWM 듀티로 평균 전압을 낮춰 사용. 모터 다수 확장 시 외부 전원(공통 GND 유지).
- **외부 전원**: 작은 부하(트랜스듀서/코인모터 1~2개)는 보드 5V(USB)로 충분. 브레드보드 전원모듈은 레일 분리·확장 시 사용.

## 6. 환경 메모
- **CH340 호환보드**: 정품이 아니라 OS가 포트를 못 잡을 수 있음 → **CH340 USB-시리얼 드라이버** 설치(Windows/macOS). 업로드/포트 인식 실패 시 **1순위 의심**.
- Arduino IDE 보드 선택은 **"Arduino Uno"**, 포트는 CH340 인식 포트.
- 시리얼 통신 기본 속도는 프로젝트 표준 **115200 baud**(`software-architecture.md §5`).

## 7. DRV8833 핀 빠른 참조
| 핀 | 의미 |
|----|------|
| VM, GND | 모터 전원(5V) / 공통 GND |
| AIN1, AIN2 | 채널 A 입력(좌) — PWM/HIGH/LOW |
| AOUT1, AOUT2 | 채널 A 출력 → 모터/트랜스듀서(좌) |
| BIN1, BIN2 | 채널 B 입력(우) |
| BOUT1, BOUT2 | 채널 B 출력(우) |
| SLP/EEP(nSLEEP) | **HIGH여야 동작**(모듈에 따라 점퍼/풀업) — 안 켜지면 확인 |

구동 패턴(채널당):
- **모터 일방향 PWM**: IN1=`analogWrite(0~255)`, IN2=LOW
- **500Hz 톤(트랜스듀서)**: IN1=`tone(pin, 500)`, IN2=LOW
- **(고급) BTL 진폭 2배**: IN1/IN2를 180° 반대 위상으로 구동
