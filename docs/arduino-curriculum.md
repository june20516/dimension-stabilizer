## 다른 세션에서 이어가는 법
1. 아래 **진행 현황** 표에서 상태가 ⬜ 또는 🔄 인 **첫 단계**를 찾는다.
2. 그 단계의 `📝 복습 노트`가 비어 있으면 = 아직 안 함. 채워져 있으면 = 완료.
3. 한 단계를 끝내면:
   - 진행 현황 표의 상태를 **✅** 로 바꾼다.
   - 그 단계 `📝 복습 노트`에 **배운 점 / 막혔던 점 / 코드 위치(스케치 파일명)** 를 1~3줄 남긴다.
   - 문서 상단 **마지막 업데이트** 날짜를 갱신한다.
4. 스케치 파일은 `firmware/` 또는 `sketches/` 아래에 단계별로 저장(예: `sketches/03_serial_parse/`).

## 0단계 — 환경 준비
- **목표**: 보드를 PC가 인식하고 첫 스케치를 올린다.
- **부품**: Uno R3 호환보드(CH340), USB 케이블.
- **개념**: Arduino IDE, 보드/포트 선택, `setup()`/`loop()`, 업로드 흐름.
- **실습**:
  1. Arduino IDE 설치.
  2. **CH340 드라이버** 설치(호환보드라 필요할 수 있음). 포트 안 잡히면 1순위 의심.
  3. 보드=`Arduino Uno`, 포트=CH340 인식 포트 선택.
  4. 예제 `File > Examples > Basics > Blink` 업로드 → 내장 LED 깜빡이면 성공.
- 🎯 **프로젝트**: 이후 모든 학습의 출발 게이트. 포트 번호를 적어두면 PC측 `config.py`(시리얼 포트)에 그대로 쓰임.
- ✅ **마일스톤**: 내장 LED가 깜빡인다.
- 📝 **복습 노트**: _(작성 전)_

## 1단계 — 디지털 출력 / 하트비트 LED
- **목표**: 외부 LED를 켜고 끈다. 이 LED를 "보드 살아있음" 표시등으로 고정.
- **부품**: LED 1개, **330Ω** 저항, 브레드보드, 점퍼.
- **개념**: `pinMode(pin, OUTPUT)`, `digitalWrite()`, `delay()`. LED 극성(긴 다리=+), 전류제한 저항 이유.
- **실습**: 디지털 핀 → 330Ω → LED 애노드, LED 캐소드 → GND. 1초 주기 깜빡임.
- 🎯 **프로젝트**: 이 LED를 **하트비트**로 약속. 이후 단계에서 "시리얼 수신 중 / 워치독 작동"을 이 LED 깜빡임으로 눈으로 확인 → 디버깅의 기본 도구.
- ✅ **마일스톤**: 외부 LED가 의도대로 깜빡인다.
- 📝 **복습 노트**: _(작성 전)_

## 2단계 — PWM = 모터 세기 (LED로 예행연습)
- **목표**: 밝기를 0~255로 연속 제어. 좌/우 두 채널을 따로 제어.
- **부품**: LED 2개(예: 초록=좌, 빨강=우) + 330Ω 각 1.
- **개념**: PWM, `analogWrite(pin, 0~255)`, Uno PWM 핀(`~` 표시: 3,5,6,9,10,11).
- **실습**: LED1 페이드(밝아졌다 어두워짐) → LED 2개를 **서로 다른 밝기**로 동시 제어.
- 🎯 **프로젝트**: 0~255 = `software-architecture.md §4`의 `level → PWM값`. **두 LED = 좌/우 모터 세기 모델**을 모터 없이 미리 완성.
- ✅ **마일스톤**: 두 LED의 밝기를 코드로 각각 다르게 줄 수 있다.
- 📝 **복습 노트**: _(작성 전)_

## 3단계 — 시리얼 프로토콜 파싱 (가장 중요한 기초)
- **목표**: PC가 보낸 텍스트 명령을 해석해 좌/우 출력에 반영.
- **부품**: 2단계 회로 그대로(코드 중심).
- **개념**: `Serial.begin(115200)`, `Serial.readStringUntil('\n')`, 문자열 → 정수 파싱(`sscanf` 또는 수동), 시리얼 모니터.
- **프로토콜**: **`L<0-255> R<0-255>\n`** (예 `L180 R0\n`) — `software-architecture.md §4` 표준.
- **실습**:
  1. 시리얼 모니터에서 받은 줄을 그대로 echo.
  2. `L`/`R` 뒤 숫자 2개 파싱 → 2단계 두 LED 밝기에 연결.
  3. 시리얼 모니터에 `L200 R50` 입력 → 좌 LED 밝게, 우 LED 어둑.
- 🎯 **프로젝트**: 이게 **PC ↔ 펌웨어 계약**. PC측 `backend_serial.py`가 붙는 지점. 잘못된/끊긴 입력 처리도 여기서 연습.
- ✅ **마일스톤**: 모니터로 친 명령대로 두 LED가 반응 = 계약 완성.
- 📝 **복습 노트**: _(작성 전)_

## 4단계 — DRV8833 + 코인모터 (Track B)
- **목표**: LED 자리를 실제 코인모터로 교체, 시리얼로 좌/우 모터 세기 제어.
- **부품**: DRV8833 모듈, 코인모터 2, (전원) 보드 5V.
- **개념**: H-브리지 드라이버, 핀 매핑(`AIN1/AIN2→AOUT`, `BIN1/BIN2→BOUT`), `nSLEEP(SLP/EEP) HIGH`, 공통 GND.
- **배선**(채널 A=좌): VM=5V, GND 공통, **AIN1=PWM 핀, AIN2=GND**, AOUT1/AOUT2→모터. 채널 B=우 동일(BIN1=PWM, BIN2=GND).
- **실습**: 3단계 코드의 LED 핀을 DRV8833 IN 핀으로 교체 → `analogWrite`로 모터 세기.
- 🎯 **프로젝트**: `hardware-design.md §B.2` 권장 회로. **(팁)** 가변저항 모듈로 모터 세기를 수동 제어해 회로부터 검증하면 시리얼 문제와 분리해 디버깅 가능.
- ✅ **마일스톤**: 시리얼 `L255 R0` → **왼쪽 모터만 진동** = Track B POC.
- 📝 **복습 노트**: _(작성 전)_

## 5단계 — tone() 500Hz 트랜스듀서 구동 (Track A)
- **목표**: 골전도 트랜스듀서를 500Hz로 구동하고 주파수를 스윕.
- **부품**: DRV8833, 골전도 트랜스듀서(8Ω) 1~2.
- **개념**: `tone(pin, freq)`, 사각파, 주파수와 체감. ⚠️ **능동 부저로는 불가**(고정 톤) → 트랜스듀서로 진행.
- **구동**: `tone(AIN1, 500)` + `AIN2=LOW`, 트랜스듀서는 AOUT1/AOUT2 (`hardware-design.md §A.5`).
- **실습**: 250 / 500 / 700 / 1000Hz 코드 스윕. 좌/우 패닝(한쪽만 구동)으로 방향 구분.
- 🎯 **프로젝트**: 이게 트랜스듀서 도착 후 **Stage 0 게이트**("두개골 진동이 느껴지는가 + 최적 위치") 검증 도구(`hardware-design.md §A.6`). 손으로 귀 뒤·유양돌기에 대보며 비교.
  - **(고급)** 힘 부족 시 AIN1/AIN2를 180° 반대 위상으로 구동(BTL) → 진폭 2배.
- ✅ **마일스톤**: 트랜스듀서를 만지면 500Hz 진동이 느껴진다.
- 📝 **복습 노트**: _(작성 전)_

## 6단계 — millis() 논블로킹 + 워치독 (안전)
- **목표**: `delay()` 없이 동작 + 패킷이 끊기면 모터를 자동으로 끈다.
- **부품**: 4단계 회로(+하트비트 LED).
- **개념**: `millis()` 논블로킹 타이밍, 마지막 수신 시각 추적, 타임아웃.
- **실습**: `lastPacketMs` 기록 → `if (millis() - lastPacketMs > TIMEOUT) { 좌우 OFF; }`. 하트비트 LED로 "살아있음" 표시.
- 🎯 **프로젝트**: `software-architecture.md §4`의 **워치독** 요구사항. PC가 죽거나 케이블이 빠졌을 때 모터가 계속 도는 사고 방지 — 펌웨어 안전 핵심.
- ✅ **마일스톤**: 시리얼 송신을 멈추면 N ms 후 모터가 자동으로 꺼진다.
- 📝 **복습 노트**: _(작성 전)_

## 7단계 — 통합 펌웨어 `firmware/dimension_stabilizer.ino`
- **목표**: 3~6단계를 한 스케치로 통합.
- **구성**: 시리얼 파싱 → 좌/우 PWM(모터) + (옵션) tone(트랜스듀서) → 워치독 → 하트비트 LED.
- **실습**: 실제 PC측 `pc/backend_serial.py`(또는 임시 송신 스크립트)와 연결 → 마우스 우측 회전 시 우측 모터.
- 🎯 **프로젝트**: `build-roadmap.md Phase 4` 완료 = **Track B 엔드투엔드 POC**.
- ✅ **마일스톤**: 마우스를 오른쪽으로 휙 → 오른쪽 모터가 회전 속도에 비례해 진동.
- 📝 **복습 노트**: _(작성 전)_

## 8단계 — 지연 측정 & 튜닝
- **목표**: 입력→진동 지연을 **수십 ms 이내**로 확인하고 감각을 다듬는다.
- **개념**: baud(115200+), 전송 주기, 라운드트립 측정.
- **실습**: 송신 주기 단축, 모터 세기 곡선·데드존 튜닝(`build-roadmap.md Phase 6`).
- 🎯 **프로젝트**: `software-architecture.md §5` 지연 목표 검증.
- ✅ **마일스톤**: 체감 지연이 거슬리지 않는 수준.
- 📝 **복습 노트**: _(작성 전)_

## 곁다리 트랙 (범용 학습 — 선택, 진행 추적 안 함)
프로젝트 임계경로는 아니지만 기초 체력에 좋음. 보유 부품으로 가능:
| 주제 | 부품 | 배우는 것 | 프로젝트 연관 |
|------|------|-----------|---------------|
| 버튼 입력·디바운싱 | 택트 스위치 | `digitalRead`, `INPUT_PULLUP`, 채터링 | 낮음 |
| 가변저항 → 모터 수동 제어 | 가변저항 모듈 | `analogRead`, `map()` | **높음**(4단계 회로 점검) |
| 자동 조명 | 조도센서(CDS)+10K 분압 | 아날로그 분압 | 낮음 |
| 거리 알람 | HC-SR04 + 능동부저 | 펄스 타이밍, `tone`/알람 | 낮음 |
| 온습도 표시 | DHT11 | 라이브러리 사용 | 낮음 |
| 서보 각도 제어 | SG90 + 가변저항 | `Servo.h`, PWM | 낮음 |
| 가스 감지 | MQ-2 | 예열·임계 | 낮음 |

> 권장: **가변저항 → 모터 수동 제어**는 4단계 직전/직후에 해두면 모터 회로를 시리얼과 분리해 검증할 수 있어 실전에 가장 유용.
