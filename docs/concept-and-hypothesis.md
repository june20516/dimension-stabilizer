# 개념 & 가설

## 1. 문제 정의
제작자는 **평면 모니터 3D 게임(FPS류)** 플레이 중 멀미/불편함을 느낀다. (VR에서는 멀미를 느끼지 않음 — 평면 게임 특유의 문제.)

평면 3D 게임에서는 머리는 정지해 있고 **마우스/컨트롤러 입력으로 화면 카메라만 회전**한다. 이때:
- **눈**: "내가 회전/이동한다"는 시각 정보 수신
- **전정기관**: "나는 정지해 있다"는 신호 수신
- → 두 신호의 **감각 충돌(sensory conflict)** 이 멀미를 유발한다고 본다.

## 2. 핵심 가설
> 화면 카메라가 도는 **방향에 맞춰** 머리에 진동을 주면, 부족한 자기운동(self-motion) 단서를 인공적으로 보충해 감각 충돌이 완화되고 멀미가 줄어든다.

예: 마우스를 오른쪽으로 휙 → 카메라 우회전 → **오른쪽 모터**가 회전 속도에 비례해 진동.

## 3. 메커니즘 후보 → 두 트랙으로 직접 비교
진동이 멀미에 작용하는 **경로**가 갈린다. 본 프로젝트는 둘 다 만들어 비교한다:

| 트랙 | 작용 경로 | 하드웨어 | 비고 |
|------|----------|----------|------|
| **A** | **전정 자극** — 속귀 운동센서에 작용 | 골전도 이어폰 500Hz | 귀 밀착 필요, 연구 충실 |
| **B** | **촉각** — 피부 방향 단서/그라운딩 | 코인 진동모터 | 위치 자유, 단순 |

- (직교 축) 신호 내용도 **방향 일치 큐** vs **노이즈 주입**으로 나뉨 — Track A 캐리어를 톤(일치 단서)↔대역 노이즈(노이즈 주입)로 바꿔 탐색 가능.
- 모터/이어폰을 **귀 뒤 유양돌기(mastoid) 근처**에 두면 전정기관과 가까워, 촉각 트랙도 일부 전정 자극을 겸할 여지가 있음.
- 어느 경로가 실제로 효과 있는지는 **3way 검증(OFF/A/B)** 으로 판별 (validation-plan.md).

## 4. 선행 연구 요약 (상세는 ../findings.md)
- 방향성 진동은 VR에서 일부 근거 (WalkingVibe n=240, 단 걷기 한정 / InertiaVibe = 본 아이디어 최근접이나 소표본).
- **평면 3D 게임 대상 방향 진동은 거의 미개척** → 본 프로젝트는 탐색적 실험.
- 노이즈 주입(GVS 등)은 효과가 일시적이고 측정도구 의존적.

## 5. 성공 기준
- **기술적 성공**: 게임 회전에 맞춰 올바른 방향 모터가 낮은 지연으로 진동한다.
- **실험적 성공(진짜 목표)**: 동일 게임/세션에서 장치 **ON일 때 OFF 대비 불편함이 체감·기록상 감소**한다. (검증 방법은 validation-plan.md)
- **명시적 실패 수용**: 효과가 없을 수 있음. "동작하는 시제품"과 "효과 입증"은 별개임을 전제로 한다.

## 6. 리스크 / 미지수
- 코인모터 진동이 뇌에 "자기운동 단서"로 binding 될지 불확실.
- 방향 일치가 오히려 새로운 충돌/주의분산을 만들 가능성.
- 지연이 크면 단서가 오히려 어긋나 역효과.
- 효과의 개인차·플라시보 → 블라인드 A/B 필요.
