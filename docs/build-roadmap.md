# 제작 로드맵 (두 트랙)

핵심 전략: **공통 코어를 먼저 만들고**, 그 위에 두 출력 트랙을 올린다. 작은 성공을 쌓는다.

## Phase 0 — 사전 준비
- 흐름 이해 (software-architecture.md §1).
- Python + `sounddevice`·`pynput`(또는 `pygame`), Arduino IDE 설치.

## Phase 1 — 장비 준비 & 구매
- **Track A**: 유선 골전도 이어폰 1개 + 별도 오디오 출력(잭/HDMI/USB 동글).
- **Track B**: 아두이노 Uno + 스타터 키트 + 코인 진동모터 2.
- 상세: hardware-design.md.

## Phase 2 — 공통 코어 (트랙 무관, 하드웨어 없이)
- 마우스 delta → 정규화 `yaw_rate` → `(left_level, right_level)` ∈ [0,1] 콘솔 출력.

✅ 마일스톤: 게임 좌우 회전 시 좌/우 level 값이 그에 맞게 출력.

## Phase 3 — Track A: 골전도 이어폰 (전정)
1. `sounddevice` 출력 장치 목록 → 이어폰 device index 찾기.
2. 그 장치에 500Hz 톤, 좌/우 패닝 테스트(왼 귀만/오른 귀만).
3. 코어 연결: `level` → 캐리어 진폭·채널 → 이어폰.

✅ 마일스톤: 오른쪽으로 휙 → 오른쪽 귀 진동 (**Track A POC**).

## Phase 4 — Track B: 아두이노 + 모터 (촉각)
1. 아두이노 기본기 — LED Blink → Fade(PWM) → 시리얼로 밝기 조절.
2. LED 자리에 모터+트랜지스터+다이오드 회로 → 시리얼로 모터 세기 제어.
3. 코어 연결: `level` → 시리얼 → 좌/우 모터.

✅ 마일스톤: 오른쪽으로 휙 → 오른쪽 모터 진동 (**Track B POC**).

> Phase 3·4는 독립 — 순서 바꾸거나 병행 가능. 부품 도착 순서대로 진행.

## Phase 5 — 컨트롤러 어댑터 (선택)
- 오른쪽 스틱 → 동일 `yaw_rate` 인터페이스로 추가.

## Phase 6 — 각 트랙 튜닝
- A: 캐리어(500Hz 톤 vs 대역 노이즈), 세기·패닝.
- B: 모터 위치(유양돌기 근처), 세기 곡선·데드존.

## Phase 7 — 3way 효과 검증
- OFF(샴) / 전정(A) / 촉각(B) 블라인드 비교 (validation-plan.md). **만들기보다 어려운 단계.**

## 난이도·시간 감각
- **Track A**: 전자공작 없음(순수 코딩). 관문 = 오디오 장치 선택·실시간 콜백.
- **Track B**: 전자공작 입문. 관문 = 배선(다이오드 방향·공통 GND)·시리얼 파싱. 납땜은 착용형 정리 때 처음.
- 공통 코어를 재사용하므로, 두 번째 트랙은 출력 백엔드만 새로 붙이면 됨.
