# 효과 검증 계획 (3way: OFF / 전정 / 촉각)

> "동작하는 시제품"과 "정말 멀미가 줄었나"는 다른 문제. 이 문서는 후자를 다룬다.
> 두 트랙(A=전정/이어폰, B=촉각/모터)을 같은 잣대로 비교하는 게 이 프로젝트의 핵심 실험.

## 0. 사전 게이트 (Stage 0 — 이 효과 검증 이전)
아래 3way 효과 검증은 "장치가 진동을 만들어낸다"가 전제다. Track A는 그 전에 **Stage 0**(트랜스듀서가 두개골 진동을 느끼게 하는가 + 최적 부착 위치)를 통과해야 한다 — 상세는 hardware-design.md A.6. 통과 못 하면(출력 부족) 트랜스듀서부터 교체.

## 1. 검증의 어려움
- **개인차·플라시보**: 기대만으로 덜 불편하게 느낄 수 있음.
- **측정 주관성**: 멀미는 정량화가 어렵다.
- **컨디션 변동**: 수면·피로·공복이 영향.
→ "켜보니 괜찮네"는 근거로 약함. 최소한의 통제 필요.

## 2. 비교 조건 (3가지)
| 조건 | 내용 |
|------|------|
| **OFF (샴)** | 이어폰·모터 착용하되 출력 비활성 (착용감 차이 제거) |
| **A = 전정** | 골전도 이어폰 500Hz 좌/우 (Track A) |
| **B = 촉각** | 코인 진동모터 좌/우 (Track B) |

→ A vs OFF, B vs OFF 로 각 효과를, A vs B 로 메커니즘 우열을 본다.

## 3. 측정 지표 (자가 기록)
SSQ(Simulator Sickness Questionnaire) 단순화, 세션마다 0~3점:
- 메스꺼움 / 어지럼 / 두통 / 눈피로 / 식은땀 / 방향감 상실
- 추가: **버틴 플레이 시간**(불편해 멈추기까지).

## 4. 실험 설계 (블라인드)
1. **동일 게임·동일 구간** 고정.
2. 세 조건(OFF/A/B)을 **무작위 순서**로 번갈아 — 한 세션당 한 조건.
3. **블라인드**: 어느 조건인지 모르게 (동거인이 무작위 설정, 또는 라벨 없는 프리셋 무작위 선택).
   - 주의: A와 B는 착용 장치가 달라 완전 블라인드가 어려움 → 최소한 "출력 ON/OFF 여부"는 모르게, 가능하면 둘 다 착용한 채 어느 쪽이 활성인지 모르게.
4. 각 세션 후 §3 지표 기록.
5. 여러 라운드 누적 후 조건별 평균 비교.

## 5. 통제할 변수
- 시간대·수면·공복 최대한 일정.
- 동일 게임·그래픽·마우스 감도, 세션 길이 고정, 세션 간 충분한 휴식.

## 6. 판정 기준 (사전 정의)
- 어떤 조건이 OFF 대비 §3 지표가 **여러 라운드 일관되게** 낮고 체감되면 → 그 트랙 "유망".
- A·B 모두 효과 없음/역효과 → 솔직히 기록하고 가설 수정:
  - 위치(유양돌기↔다른 부위), 캐리어(500Hz 톤↔노이즈), 매핑(지연·세기·곡선) 변경.
- **명시적 실패 수용**: 둘 다 효과 없을 수 있으며, 그 결과도 유효한 결론.

## 7. 기록 양식 (예시)

|------|--------|-------------|------|----------|--------|------|--------|----------|------|
|      |        |             |      |          |        |      |        |          |      |

> 실제 조건(OFF/A/B) 정답은 별도 파일에 보관, 점수 기록 후 매칭.
